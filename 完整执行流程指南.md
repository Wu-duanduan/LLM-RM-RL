# 完整执行流程指南 v2.1

## 📋 完整训练流程

```
【可选】阶段 0: 高级语义处理 (轨迹采样 + LTL 归纳)
    ↓
```

## 🎯 两种运行模式

### 模式 1：标准模式（推荐，快速）
- 使用 Claude 直接生成 TRM/SRM 规则
- 适合快速原型和数据收集
- 运行时间：8-16 小时

### 模式 2：高级模式（研究用，慢速）
- 启用轨迹采样 + LTL 归纳 + 贝叶斯发现
- 适合复杂任务和学术研究
- 运行时间：16-32 小时

## 🆕 阶段 0：高级语义处理（可选）

### 何时使用高级模式？

**使用标准模式**（推荐）：
- ✅ 快速原型开发
- ✅ 数据收集和预训练
- ✅ 任务相对简单

**使用高级模式**：
- ✅ 复杂多分支任务
- ✅ 需要形式化验证
- ✅ 学术研究和论文

### 启用高级模式

修改 [`disaster_env.py`](disaster_env.py:52) 第 52 行：

```python
# 标准模式（默认）
self.brain = LLMStrategicBrain(MODEL_PATH, use_advanced_processing=False)

# 高级模式
self.brain = LLMStrategicBrain(MODEL_PATH, use_advanced_processing=True)
```

## 🔄 MARL 控制模式详解

### 控制模式对比

| 特性 | Gurobi 专家模式 | MARL 控制模式 |
|------|----------------|---------------|
| **use_marl_control** | False | True |
| **决策来源** | Gurobi 优化器 | MARL 策略网络 |
| **Gurobi 调用** | ✅ 调用 | ❌ 跳过 |
| **任务设置** | 自动（Gurobi） | 手动（set_marl_task） |
| **用途** | 数据收集 | 在线训练/部署 |
| **适用阶段** | 阶段 1 | 阶段 3-5 |

### MARL 控制模式工作流

```python
# 1. 创建环境（启用 MARL 控制）
env = DisasterEnv(
    ...,
    use_marl_control=True  # 🔴 关键参数
)

# 2. MARL 模型选择任务
entity_idx = marl_model.forward(obs)

# 3. 设置任务
env.set_marl_task('Dog_Alpha', task_id)

# 4. 执行步进（跳过 Gurobi）
obs, reward, done, info = env.step(actions)
# 输出: [模块C] 🤖 MARL 控制模式 - 跳过 Gurobi 调用

# 5. 任务完成后清除
if task_completed:
    env.clear_marl_task('Dog_Alpha')
```

## ⚠️ 常见问题

### Q1: Gurobi 服务连接失败
```
❌ [模块C] 教师网络失联: Connection refused
```
**解决**：
- 检查 Windows 主机上的 Gurobi 服务是否运行
- 仅在数据收集阶段需要（use_marl_control=False）
- 在线训练阶段不需要 Gurobi（use_marl_control=True）

### Q2: MARL 模式下任务一直失败
```
❌ Dog_Alpha 任务失败 | 奖励: -5.00
```
**原因**：
- 模型未经训练（随机初始化）
- 需要通过训练逐步学习

**解决**：
- 加载预训练模型：`--load_pretrain`
- 增加训练 episodes
- 调整学习率和奖励权重

### Q3: 维度不匹配错误
```
RuntimeError: mat1 and mat2 shapes cannot be multiplied (1x36 and 108x256)
```
**解决**：
- 检查 `global_state_dim` 配置
- 应该是：3个智能体×4维 + 16维任务 + 8维约束 = 36维
- 已在 train_mappo.py 中修复

### Q4: 预训练准确率低于 70%
**解决**：
- 增加训练数据量
- 增加训练 epochs
- 调整学习率

### Q5: 在线微调奖励不增长
**解决**：
- 检查奖励函数权重
- 调整 PPO 超参数
- 增加探索率
- 确认 MARL 控制模式已启用
